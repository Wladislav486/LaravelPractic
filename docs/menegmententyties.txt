Задание
Создать фун-ал управления категориями/постами/тегами - CRUD
1 Просмотр сущности не нужен
2 В правом меню сделать пункт сущностей с выпадающим списком
- ссылка на список сущностей
- ссылка на создание сущности
3 при переходе на список сущностей выводить сущность с ссылками на
- изменение
- удаление
над списком вывести кнопку добавить сущность
4 данные находятся в соответствующей таблице
- работу вести с ней


Решение
1 Создаём контроллер ресурсов
2 Создаём маршрут для контроллера ресурсов в группе админ
3 создаём необходимые представления
4 Корректируем вёрстку
- https://adminlte.io/themes/v3/pages/tables/simple.html
- копируем необходимые html сущности с данного сайта и корректируем вёрстку
- создаём формы, таблицу
- контейнеры для вывода сообщений
5 выводим маршруты контроллера в консоле
6 прописываем маршруты для ссылок и форм
7 реализуем логику в контроллере
- валидация
- запросы через orm к бд
- флеш сообщения
8 добавляем вывод пагинации


Доп
1 Просмотреть все маршруты
php artisan route:list
2 Просмотреть машруты по маске
php artisan route:list --path=admin/cat
3 логику валидации можно вынести в отдельный класс
php artisan make:request StoreCategory

Дополнительная логика постов
Отличия
- картинка
- связь могие ко многим с тегами
1) Добавление поста
- загрузка файлов на сайт через форму
    - по умолчанию используется disc - local (настройка)
    - в теории сделали симлинк в папке public
    - В практике поменяем путь в настройке в файле config/filesystems.php
- Сохранение связи многие ко многим
    https://laravel.su/docs/7.x/eloquent-relationships#updating-belongs-to-relationships
    пример
    $user->roles()->sync([1, 2, 3]);
    старые привязки удаляет, новые добавляет
3) вывод тегов в читаемом виде в списке в админке, только тайтлы
- есть json
- работа с коллекциями
4) изменение поста
- меняем плейсхолдеры на значения
- в экшене вывода формы на изменение получаем пост категории и теги поста
- вставляем значения во въюшке
- в экшене обработки отправки формы
    - обновление изображения
    - обновление поста
    - обновления привязок к тегам
5) Удаление
- файлов изображений
- привязок к тегам
